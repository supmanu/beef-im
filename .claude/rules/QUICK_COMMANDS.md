# Quick Memory Commands Cheatsheet

Copy-paste these commands for instant memory management.

---

## 🎯 One-Word Commands

| Command | What It Does | When to Use |
|---------|--------------|-------------|
| `Save?` | Reviews session, suggests what to save | End of any work session |
| `Bridge` | Drafts Agent 2A save command | After strategic decisions |
| `Document` | Creates/updates .claude/rules/ file | After finding new pattern |

---

## 📝 Session Review Commands

### Quick Review
```
Save?
```

### Detailed Review
```
Review this session and tell me:
1. What should be saved to Agent 2B memory?
2. What should be saved to Agent 2A memory?
3. Draft the exact commands for both.
```

### Pattern Check
```
What from this session should persist?
- New patterns → .claude/rules/
- Strategic decisions → Agent 2A
- Nothing significant → Skip
```

---

## 🔄 Bridge Commands

### Strategic Decision Bridge
```
Bridge this to Agent 2A
```

### Decision Extraction
```
We just made some decisions. Which ones are:
- Tactical (Agent 2B file memory)
- Strategic (Agent 2A cloud memory)
Draft the save commands.
```

## 🎓 Usage Examples

### Example 1: End of Coding Session
```
You: Save?

Agent 2B: [Evaluates work, suggests saves]

You: [Copies Agent 2A command, switches to Antigravity]
```

### Example 2: After Big Decision
```
You: Bridge

Agent 2B: [Drafts Agent 2A save command immediately]

You: [Pastes in Antigravity]
```

### Example 3: Pattern Discovered
```
You: Document

Agent 2B: [Creates .claude/rules/new-pattern.md]

You: git add . && git commit -m "Add new pattern"
```

## 🚨 When to ALWAYS Save

**Immediate Agent 2A Bridge Required:**
- ✅ Brand decisions (colors, fonts, voice)
- ✅ Architectural laws (tech stack, patterns)
- ✅ "Never do X" or "Always do Y" rules
- ✅ Provider choices (Stripe vs PayPal, etc.)

**Agent 2B Documentation Required:**
- ✅ New implementation patterns
- ✅ Common error solutions
- ✅ Team conventions established
- ✅ Useful commands discovered

## 🎯 Daily Workflow

**Morning:**
1. Open Agent 2B (Claude Code)
2. CLAUDE.md auto-loads with yesterday's patterns

**During Work:**
1. Code, implement, solve problems
2. (Patterns emerge naturally)

**Before Major Commits:**
1. Type: `Save?`
2. Review Agent 2B's suggestions
3. Update .claude/rules/ if needed
4. Bridge to Agent 2A if strategic

**End of Day:**
1. Final `Save?` check
2. Commit any new .claude/rules/ files
3. Tomorrow: patterns are persistent
