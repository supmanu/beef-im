## 🚀 How to Use

### Daily Workflow

1. **Start work in Agent 2B (this terminal):**
   ```
   Code, refactor, implement features
   Document patterns in .claude/rules/ as you go
   ```

2. **Access memory anytime:**
   ```
   /memory                    # Edit CLAUDE.md
   @.claude/rules/payload.md  # Reference specific patterns
   ```

3. **Bridge strategic decisions to Agent 2A:**
   - Switch to Antigravity Chat
   - Manually save important decisions to MCP memory
   - Use Claude Sonnet 4.5 for memory operations

### Memory Commands

```bash
/memory                  # Edit project memory files
@package.json            # Reference any file
@.claude/rules/nextjs.md # Import specific rules
```

## ❌ What We Uninstalled

**claude-mem plugin v7.2.4** - Removed due to:
- "ProcessTransport is not ready" crashes on Windows
- Chroma sync failures
- Unreliable archive creation
- Zero archives generated despite multiple sessions

**Decision:** Built-in Claude Code memory is more reliable, portable, and version-controlled.
