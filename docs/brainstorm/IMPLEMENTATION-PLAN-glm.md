## Format System (Visual Metadata)

Format determines the article page's visual template + header badge. It is NOT a URL segment.

| Format | Header Badge | Layout | Best For |
|---|---|---|---|
| `forensic` | `CASE FILE` | Myth → Evidence → Mechanism → Verdict structure | Insurance teardowns, deep meat investigations |
| `experiment` | `EXPERIMENT LOG` | Setup → Variables → Observations → Results → Limitations | Structured tests, comparisons |
| `field-note` | `FIELD NOTE` | Minimal, timestamp-forward, single column | Quick observations, tips |
| `longform` | (none — default) | Standard prose, single column | Meat techniques, personal essays, general notes |

### Step 8 — Visual Foundation (NEW — stolen effects)

This step installs the CSS-first effects base that all subsequent phases build on.

#### 8a. Font Metric Fallbacks (from Thorgal)

Thai fonts have extreme vertical metric differences between the web font and fallback. Without `size-adjust` / `ascent-override` / `descent-override`, the first paint shows content at wrong positions, then reflows when the font loads — visible CLS.

Add fallback `@font-face` declarations for each of the 4 font families in `src/styles/global.css`:

```css
@font-face {
  font-family: "Anuphan-fallback";
  src: local("Noto Sans Thai");
  size-adjust: 103.2%;
  ascent-override: 95%;
  descent-override: 25%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "Sarabun-fallback";
  src: local("Noto Sans Thai");
  size-adjust: 106.1%;
  ascent-override: 92%;
  descent-override: 28%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "K2D-fallback";
  src: local("Tahoma");
  size-adjust: 104.5%;
  ascent-override: 88%;
  descent-override: 32%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "IBM Plex Mono-fallback";
  src: local("Courier New");
  size-adjust: 97.8%;
  ascent-override: 96%;
  descent-override: 22%;
  line-gap-override: 0%;
}
```

Update Tailwind `@theme` / `fontFamily` tokens to include fallback chain:
```css
--font-display: "Anuphan", "Anuphan-fallback", sans-serif;
--font-thai: "Sarabun", "Sarabun-fallback", sans-serif;
--font-hand: "K2D", "K2D-fallback", sans-serif;
--font-mono: "IBM Plex Mono", "IBM Plex Mono-fallback", monospace;
```

**Note:** Exact `size-adjust` / `ascent-override` / `descent-override` values must be calibrated using [Font Style Matcher](https://s3.amazonaws.com/ftlabs-font-style-matcher/index.html) or equivalent, comparing each web font against its chosen local fallback. The values above are estimates — measure real CLS reduction using Lighthouse or `web-vitals` before/after.

#### 8b. Noise Grain Overlay (from Pixila)

Generate a 100x100px grayscale noise texture tile as `public/noise.webp` (~2KB). Add in `global.css`:

```css
.nb-page {
  position: relative;
}

.nb-page::after {
  content: '';
  position: fixed;
  inset: 0;
  background: url('/noise.webp') repeat;
  opacity: 0.06;
  mix-blend-mode: multiply;
  pointer-events: none;
  z-index: 9998;
}
```

This adds paper grain over the existing cream background and grid lines. At 6% opacity with `multiply` blend, it's perceptible but not distracting. The `fixed` positioning ensures grain stays in place during scroll. `pointer-events: none` makes it fully click-through.

#### 8c. @property Custom Properties (from Pixila — the foundational unlock)

Register two animatable custom properties at `:root` level in `global.css`:

```css
@property --ink-spread {
  syntax: '<percentage>';
  inherits: true;
  initial-value: 100%;
}

@property --paper-offset {
  syntax: '<percentage>';
  inherits: true;
  initial-value: 0%;
}
```

These enable CSS to animate `clip-path` polygon expressions through `calc()` — without `@property`, CSS cannot interpolate custom properties, so `clip-path: polygon(calc(50% + var(--ink-spread)))` would jump instead of animate.

**Browser support:** Chrome 85+, Edge 85+, Safari 15.4+, Firefox 128+. Firefox < 128 falls back to content rendering normally (no entrance animation, which is acceptable progressive enhancement).

#### 8d. mix-blend-difference Nav (from Thorgal)

Add to `.nb-sticky-nav` in `global.css`:

```css
.nb-sticky-nav {
  /* existing styles... */
  mix-blend-mode: difference;
}
```

This makes the sticky navigation text automatically contrast against whatever background it sits over — cream hero, white article body, or any future colored section. The text color inverts based on underlying content, eliminating the need for JavaScript scroll-based color inversion.

**Note:** `mix-blend-mode: difference` requires the nav to use a neutral/text color that benefits from inversion. Test that `--color-ink` (near-black) and date/label text invert correctly against cream (`#F0EADC`) and any colored backgrounds.

#### 8e. clip-path Wipe Keyframes (from Thorgal)

Define reusable `clip-path` wipe animation keyframes in `global.css`:

```css
@keyframes nb-wipe-up {
  from { clip-path: inset(0 0 100% 0); }
  to   { clip-path: inset(0 0 0 0); }
}

@keyframes nb-wipe-down {
  from { clip-path: inset(100% 0 0 0); }
  to   { clip-path: inset(0 0 0 0); }
}

@keyframes nb-wipe-right {
  from { clip-path: inset(0 0 0 100%); }
  to   { clip-path: inset(0 0 0 0); }
}

/* For badges and stamps — horizontal reveal */
@keyframes nb-stamp-reveal {
  from { clip-path: inset(0 100% 0 0); }
  to   { clip-path: inset(0 0 0 0); }
}

/* Progressive enhancement: skip for reduced motion */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

These keyframes will be used by NotebookEntry scroll reveals, format badges, and stamp elements in Phase 1.

#### 8f. Dual-Layer Hover Utilities (from Pixila)

Add CSS utilities for the two-layer clip-path hover effect in `global.css`:

```css
.nb-dual-hover {
  position: relative;
  overflow: hidden;
}

.nb-dual-hover-top,
.nb-dual-hover-bottom {
  grid-area: stack;
  display: block;
  transition: clip-path 350ms ease, transform 350ms ease;
}

.nb-dual-hover-top {
  clip-path: inset(0 0 0 0);
}

.nb-dual-hover-bottom {
  clip-path: inset(25px 0 0 0);
  transform: translateY(12px) skewY(2deg);
}

.nb-dual-hover:hover .nb-dual-hover-top {
  clip-path: inset(0 0 25px 0);
  transform: translateY(-12px) skewY(-2deg);
}

.nb-dual-hover:hover .nb-dual-hover-bottom {
  clip-path: inset(0 0 0 0);
  transform: translateY(0) skewY(0);
}
```

Usage: Wrap two `<span>` elements with the same text inside a `.nb-dual-hover` parent. Top layer is the visible default, bottom layer slides in from below on hover with a slight mechanical skew. Duration: 350ms per Pixila's timing.

### 2d. Ink Spread Page Reveal (NEW — stolen from Pixila)

The flagship entrance effect. When a reader navigates to any article page for the first time in a session, the content reveals through an asymmetric ink-spread polygon animation. On repeat navigation within the same session, the animation is skipped.

**Design:** The polygon starts from a center point (like an ink drop hitting paper) and expands outward asymmetrically — wider horizontally than vertically, with organic non-uniform timing. The shape evokes ink seeping into paper fibers rather than the sci-fi amoeba of the original Pixila effect.

**CSS in `global.css`:**
```css
/* Ink Spread — article entrance reveal */
/* Requires @property --ink-spread (registered in Phase 0 Step 8c) */

@keyframes ink-reveal {
  from {
    --ink-spread: 100%;
  }
  to {
    --ink-spread: 0%;
  }
}

.nb-ink-reveal {
  clip-path: polygon(
    /* Center collapse point — all vertices converge when --ink-spread = 100% */
    /* When --ink-spread = 0%, polygon becomes full rectangle (inset: 0) */
    calc(50% - var(--ink-spread) * 0.52)  calc(50% - var(--ink-spread) * 0.48),
    calc(50% + var(--ink-spread) * 0.48)  calc(50% - var(--ink-spread) * 0.45),
    calc(50% + var(--ink-spread) * 0.55)  calc(50% - var(--ink-spread) * 0.10),
    calc(50% + var(--ink-spread) * 0.50)  calc(50% + var(--ink-spread) * 0.50),
    calc(50% + var(--ink-spread) * 0.42)  calc(50% + var(--ink-spread) * 0.55),
    calc(50% - var(--ink-spread) * 0.45)  calc(50% + var(--ink-spread) * 0.52),
    calc(50% - var(--ink-spread) * 0.50)  calc(50% + var(--ink-spread) * 0.35),
    calc(50% - var(--ink-spread) * 0.53)  calc(50% - var(--ink-spread) * 0.05)
  );
  animation: ink-reveal 0.8s ease-out forwards;
  will-change: clip-path;
}

/* Progressively skip for browsers that don't support @property */
@supports not (at-rule: @property) {
  .nb-ink-reveal {
    clip-path: none;
    animation: none;
  }
}
```

**JavaScript in `ArticleLayout.astro` (inline, ~200 bytes):**
```js
(function() {
  var K = 'beef_ink';
  var m = document.querySelector('main');
  if (!m || sessionStorage.getItem(K)) {
    if (m) m.style.clipPath = 'none';
    return;
  }
  m.classList.add('nb-ink-reveal');
  m.addEventListener('animationend', function() {
    m.style.clipPath = 'none';
    m.classList.remove('nb-ink-reveal');
    sessionStorage.setItem(K, '1');
  }, { once: true });
  // Fallback: skip after 2s if animation fails
  setTimeout(function() {
    if (m.classList.contains('nb-ink-reveal')) {
      m.style.clipPath = 'none';
      m.classList.remove('nb-ink-reveal');
      sessionStorage.setItem(K, '1');
    }
  }, 2000);
})();
```

**Timing:** 0.8s with `ease-out` — feels like a drop of ink blooming on paper. The `ease-out` timing means the initial expansion is fast (ink soaks quickly) and decelerates as it reaches the edges (ink saturates paper). Session-aware (skips on repeat navigation).

**The polygon coefficients** (the `0.52`, `0.48`, etc.) create slight asymmetry — the "ink" doesn't spread in a perfect circle. These values should be visually tuned during implementation. The general principle: wider horizontal spread (0.52-0.55) than vertical (0.45-0.50) to suggest capillary action through paper fibers.

**Note:** `calc()` with `var()` inside `clip-path: polygon()` only animates when the custom property is registered via `@property` with a `<percentage>` syntax. This is the Pixila technique's core innovation.

## Effects Research Reference

For future reference, the primary techniques stolen and their sources:

### From Thorgal (thorgal.com)

Built with **Astro v6.1.8**. 12 custom elements (Web Components) with selective island hydration. Each component is a separate Astro island with its own `<script type="module">`. Uses **Taxi** for page transitions.

1. **Metric-compatible font fallbacks** — `@font-face` fallback families using `size-adjust`, `ascent-override`, `descent-override`, `line-gap-override` on local("Arial") to create CLS-free font loading. Thorgal does this for "Artifex Hand CF" and "Mulish" with precise percentage values.

2. **Character-by-character text reveal** — `<c-text-reveal>` custom element with `data-animation="char"` and `data-stagger="0.01"` (10ms per character). Replays each time section becomes active via `data-repeat="true"`. Used on section headings.

3. **clip-path inset wipe-up badges** — `<c-badge>` uses `clip-path: inset(0 0 100% 0)` initial state, animating to `inset(0 0 0 0)` over 0.5s. Creates a vertical bottom-up reveal.

4. **mix-blend-difference navigation** — Fixed sidebar dot navigation and scroll indicator use `mix-blend-difference` to auto-contrast against the underlying 3D canvas content. Eliminates need for scroll-position-aware JS color switching.

5. **Vertical section snap via CSS custom properties** — `--door-index` custom property drives `translate-y-[calc(var(--door-index,0)*-100svh)]` on the main content wrapper, creating fullscreen-scroll sections controlled by JS but computed via CSS. No scroll-snap library.

6. **ScaleY progress bars** — Loader uses `transform: scaleY(0)` with `origin-top` / `origin-bottom` for vertical fill animations, plus `tabular-nums` for stable number counter widths.

### From Pixila (pixila.net)

Built with **Astro v5.17.3**. Purely CSS-first approach with minimal JS. **No Three.js, WebGL, or canvas elements at all** — impressive visual impact through CSS alone. Uses **Partytown** to offload Google Analytics to a Web Worker.

1. **@property + clip-path polygon page reveal** — The flagship technique. `CSS.registerProperty({ name: "--offset", syntax: "<percentage>", inherits: true, initialValue: "14%" })` enables CSS to animate a custom property through `calc()` expressions inside `clip-path: polygon()`. The page reveals through a 10-point asymmetric polygon that animates from a narrow slit to full coverage over 0.7s. **This is the single most impactful technique** — it creates a cinematic page entrance with zero-vocabulary CSS.

2. **Dual-layer clip-path text hover** — Two stacked `<span>` elements in a CSS Grid. Top layer starts at `clip-path: inset(0 0 0 0)`, bottom layer starts at `clip-path: inset(25px 0 0 0)` offset by `translateY(12px) skewY(2deg)`. On hover, top layer clips away bottom-up while skewing, bottom layer reveals top-down with de-skew. 350ms timing. Creates a mechanical/instrument text swap feel.

3. **SVG stroke-dashoffset annotation pointers** — Three-part staggered CSS animation system:
   - Diamond (losange): `0.25s blink` then `0.2s scale-in` with `cubic-bezier(0.175, 0.885, 0.32, 1.5)` (spring overshoot)
   - Line: `0.15s blink` at 0.3s delay, then `1s stroke-dashoffset` from 250 to 0
   - Text: `0.25s blink` at 0.75s delay
   - Reverse animation for leaving state with inverted timing.

4. **Noise texture overlay** — Full-page `noise.webp` at 8% opacity with `mask: linear-gradient(90deg, transparent, white 20%, white 80%, transparent)` and `mix-blend-mode: multiply`. Adds film grain / paper texture without a separate image request per page. Pixila uses 8% on a dark theme; beef.im should use 6% on cream.

5. **Ghost text layer (15% echo)** — `pixila-text` component renders text twice: once at full opacity and once at `opacity: 0.15`, both stacked via CSS Grid. Creates a typographic echo/shadow without blur. beef.im adapts this to 8% opacity with `translateY(4px)` offset to simulate ink bleed-through from the reverse side of paper.

6. **Session-aware preloader** — `sessionStorage.getItem("pixila_preloaded")` skips the preloader on revisits within the same session. Pure UX win, no downside. beef.im applies this to the ink-reveal animation.

7. **Rotating losange (diamond) button** — A `-rotate-45` diamond with `border-2` that transitions to `rotate-45` on hover (90-degree flip). Not adapted for beef.im (doesn't fit the notebook metaphor) but recorded for reference.

8. **CSS-only marquee with viewport pausing** — `@keyframes scroll { from { transform: translate(0) } to { transform: translate(-25%) } }` with `.marquee-container:not(.in-viewport) .marquee-track { animation-play-state: paused }` via IntersectionObserver. Not adapted (beef.im has no scrolling logos), but the technique is noted for potential future use in a "latest entries" ticker.
