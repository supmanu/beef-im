## Philosophy: "Zero JS on Content, Editorial Moments Get the Juice"

The zero-JS default applies strictly to **article body pages** — readers should read without JavaScript. The **surfaces around the content** (hero, transitions, hub pages, tools) are where editorial personality lives. Those use targeted JS/WebGL without betraying the notebook aesthetic.

| Surface | JS Budget | Rationale |
|---|---|---|
| Article body (`/insurance/<slug>/`, `/meat/<slug>/`) | **Zero** | Text renders without JS. Binding Thread is the only exception (~500 B inline). |
| Homepage hero + flat list | **Lightweight** | Session-aware skip, ghost echo, text decode, scroll-reveal, grain overlay. |
| Page transitions (home → article, article → article) | **Lightweight** | CSS Houdini ink spread + Astro View Transitions API fallback. Content stays anchored. |
| Hub pages (`/insurance`, `/meat`, `/archive`) | **Medium** | Horizontal scrolls, animated filters, marquees, pause-on-offscreen, mix-blend nav. |
| Tools (`/tools/<slug>`) | **React Island** | Canvas charts, interactive calculators — already in plan. |
| Dedicated "experience" pages (future) | **Full** | WebGL shaders — isolated to specific moments. |

## Format System (Visual Metadata)

Format determines the article page's visual template + header badge. It is NOT a URL segment.

| Format | Header Badge | Layout | Best For |
|---|---|---|---|
| `forensic` | `CASE FILE` | Myth → Evidence → Mechanism → Verdict structure | Insurance teardowns, deep meat investigations |
| `experiment` | `EXPERIMENT LOG` | Setup → Variables → Observations → Results → Limitations | Structured tests, comparisons |
| `field-note` | `FIELD NOTE` | Minimal, timestamp-forward, single column | Quick observations, tips |
| `longform` | (none — default) | Standard prose, single column | Meat techniques, personal essays, general notes |

All four formats share the same MDX component library (Highlight, MarginNote, ScrapCard, CorrectionBlock, VerdictSeal). The format controls the page structure wrapping around the MDX body.

#### 8a. Font Metric Fallbacks (from Thorgal) — GLM

Thai fonts have extreme vertical metric differences between the web font and fallback. Without `size-adjust` / `ascent-override` / `descent-override`, the first paint shows content at wrong positions, then reflows when the font loads — visible CLS.

Add fallback `@font-face` declarations for each of the 4 font families in `src/styles/global.css`:

```css
@font-face {
  font-family: "Anuphan-fallback";
  src: local("Noto Sans Thai");
  size-adjust: 103.2%;
  ascent-override: 95%;
  descent-override: 25%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "Sarabun-fallback";
  src: local("Noto Sans Thai");
  size-adjust: 106.1%;
  ascent-override: 92%;
  descent-override: 28%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "K2D-fallback";
  src: local("Tahoma");
  size-adjust: 104.5%;
  ascent-override: 88%;
  descent-override: 32%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "IBM Plex Mono-fallback";
  src: local("Courier New");
  size-adjust: 97.8%;
  ascent-override: 96%;
  descent-override: 22%;
  line-gap-override: 0%;
}
```

Update Tailwind `@theme` / `fontFamily` tokens to include fallback chain:
```css
--font-display: "Anuphan", "Anuphan-fallback", sans-serif;
--font-thai: "Sarabun", "Sarabun-fallback", sans-serif;
--font-hand: "K2D", "K2D-fallback", sans-serif;
--font-mono: "IBM Plex Mono", "IBM Plex Mono-fallback", monospace;
```

**Note:** Exact `size-adjust` / `ascent-override` / `descent-override` values must be calibrated using [Font Style Matcher](https://s3.amazonaws.com/ftlabs-font-style-matcher/index.html) or equivalent, comparing each web font against its chosen local fallback. The values above are estimates — measure real CLS reduction using Lighthouse or `web-vitals` before/after.

#### 8b. Noise Grain Overlay (from Pixila) — GLM + Kimi

Generate a 128×128px grayscale noise texture tile as `public/noise.webp` (~2KB). Add in `global.css`:

```css
body::after {
  content: '';
  position: fixed;
  inset: 0;
  background: url('/noise.webp') repeat;
  background-size: 128px;
  opacity: 0.06;
  mix-blend-mode: multiply;
  pointer-events: none;
  z-index: 9999;
}
@media (prefers-reduced-motion: reduce) {
  body::after { display: none; }
}
```

This adds paper grain over the existing cream background and grid lines. At 6% opacity with `multiply` blend, it's perceptible but not distracting. The `fixed` positioning ensures grain stays in place during scroll. `pointer-events: none` makes it fully click-through. `prefers-reduced-motion` disables it for users who prefer reduced motion.

#### 8c. @property Custom Properties (from Pixila — the foundational unlock) — GLM

Register two animatable custom properties at `:root` level in `global.css`:

```css
@property --ink-spread {
  syntax: '<percentage>';
  inherits: true;
  initial-value: 100%;
}

@property --paper-offset {
  syntax: '<percentage>';
  inherits: true;
  initial-value: 0%;
}
```

These enable CSS to animate `clip-path` polygon expressions through `calc()` — without `@property`, CSS cannot interpolate custom properties, so `clip-path: polygon(calc(50% + var(--ink-spread)))` would jump instead of animate.

**Browser support:** Chrome 85+, Edge 85+, Safari 15.4+, Firefox 128+. Firefox < 128 falls back to content rendering normally (no entrance animation, which is acceptable progressive enhancement).

### 2d. Ink Spread Page Reveal (NEW — stolen from Pixila) — GLM

The flagship entrance effect. When a reader navigates to any article page for the first time in a session, the content reveals through an asymmetric ink-spread polygon animation. On repeat navigation within the same session, the animation is skipped.

**Design:** The polygon starts from a center point (like an ink drop hitting paper) and expands outward asymmetrically — wider horizontally than vertically, with organic non-uniform timing. The shape evokes ink seeping into paper fibers rather than the sci-fi amoeba of the original Pixila effect.

**CSS in `global.css`:**
```css
/* Ink Spread — article entrance reveal */
/* Requires @property --ink-spread (registered in Phase 0 Step 8c) */

@keyframes ink-reveal {
  from {
    --ink-spread: 100%;
  }
  to {
    --ink-spread: 0%;
  }
}

.nb-ink-reveal {
  clip-path: polygon(
    /* Center collapse point — all vertices converge when --ink-spread = 100% */
    /* When --ink-spread = 0%, polygon becomes full rectangle (inset: 0) */
    calc(50% - var(--ink-spread) * 0.52)  calc(50% - var(--ink-spread) * 0.48),
    calc(50% + var(--ink-spread) * 0.48)  calc(50% - var(--ink-spread) * 0.45),
    calc(50% + var(--ink-spread) * 0.55)  calc(50% - var(--ink-spread) * 0.10),
    calc(50% + var(--ink-spread) * 0.50)  calc(50% + var(--ink-spread) * 0.50),
    calc(50% + var(--ink-spread) * 0.42)  calc(50% + var(--ink-spread) * 0.55),
    calc(50% - var(--ink-spread) * 0.45)  calc(50% + var(--ink-spread) * 0.52),
    calc(50% - var(--ink-spread) * 0.50)  calc(50% + var(--ink-spread) * 0.35),
    calc(50% - var(--ink-spread) * 0.53)  calc(50% - var(--ink-spread) * 0.05)
  );
  animation: ink-reveal 0.8s ease-out forwards;
  will-change: clip-path;
}

/* Progressively skip for browsers that don't support @property */
@supports not (at-rule: @property) {
  .nb-ink-reveal {
    clip-path: none;
    animation: none;
  }
}
```

**JavaScript in `ArticleLayout.astro` (inline, ~200 bytes):**
```js
(function() {
  var K = 'beef_ink';
  var m = document.querySelector('main');
  if (!m || sessionStorage.getItem(K)) {
    if (m) m.style.clipPath = 'none';
    return;
  }
  m.classList.add('nb-ink-reveal');
  m.addEventListener('animationend', function() {
    m.style.clipPath = 'none';
    m.classList.remove('nb-ink-reveal');
    sessionStorage.setItem(K, '1');
  }, { once: true });
  // Fallback: skip after 2s if animation fails
  setTimeout(function() {
    if (m.classList.contains('nb-ink-reveal')) {
      m.style.clipPath = 'none';
      m.classList.remove('nb-ink-reveal');
      sessionStorage.setItem(K, '1');
    }
  }, 2000);
})();
```

**Timing:** 0.8s with `ease-out` — feels like a drop of ink blooming on paper. The `ease-out` timing means the initial expansion is fast (ink soaks quickly) and decelerates as it reaches the edges (ink saturates paper). Session-aware (skips on repeat navigation).

**The polygon coefficients** (the `0.52`, `0.48`, etc.) create slight asymmetry — the "ink" doesn't spread in a perfect circle. These values should be visually tuned during implementation. The general principle: wider horizontal spread (0.52-0.55) than vertical (0.45-0.50) to suggest capillary action through paper fibers.

**Note:** `calc()` with `var()` inside `clip-path: polygon()` only animates when the custom property is registered via `@property` with a `<percentage>` syntax. This is the Pixila technique's core innovation.

#### 2f-1. Text Scramble / Decode (Pixila)

**What:** Category stamps and format badges (e.g., `CASE FILE`, `EXPERIMENT LOG`) initially appear as scrambled random characters (`!<>-_/[]{}—=+*^?#________`), then mechanically decode to the final text. Like a teletype or declassified document.

**Where:** Homepage format badges, hub page titles, article header stamps.

**Tech:** Vanilla JS class (~1 KB). Uses `requestAnimationFrame` to cycle characters before settling. Triggered on `IntersectionObserver` (appears when scrolled into view) or page load.

**Implementation:** `src/components/TextScramble.astro` — reusable Astro component.

```js
// ~1 KB vanilla JS
class TextScramble {
  constructor(el) {
    this.el = el;
    this.chars = '!<>-_/[]{}—=+*^?#________';
    this.original = el.textContent;
  }
  decode() {
    const length = this.original.length;
    let iter = 0;
    const tick = () => {
      this.el.textContent = this.original
        .split('')
        .map((char, i) => {
          if (i < iter) return this.original[i];
          return this.chars[Math.floor(Math.random() * this.chars.length)];
        })
        .join('');
      if (iter < length) {
        iter += 1 / 3;
        requestAnimationFrame(tick);
      }
    };
    tick();
  }
}
```

**Cost:** ~1 KB JS. Only loaded on pages that use it (homepage, hubs).

**Guardrail:** If it feels gimmicky on editorial stamps after deployment, remove `TextScramble.astro` from stamps and keep only for calculator results (Phase 4c).

### 4d. WebGL Warmth Layer (Pixila — Guardrail: Deferred, Reversible)

**What:** A single full-screen fragment shader on the hero background only. Slow warm candlelight flicker over parchment, or subtle paper fiber texture. Not a video, not a CSS gradient — a living, breathing light layer.

**Where:** Homepage hero background only. Behind the notebook grid, above the parchment base.

**Tech:** Raw WebGL (not Three.js). Single `<canvas>` element with a fragment shader. ~3 KB.

**Why raw WebGL, not Three.js:** Three.js adds ~150 KB. A single fullscreen quad with a fragment shader is ~50 lines of WebGL boilerplate + the shader itself. No library needed.

**Shader concept (GLSL):**
```glsl
// Simplex noise-based warm flicker
uniform float u_time;
uniform vec2 u_resolution;

void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  float noise = snoise(vec3(uv * 3.0, u_time * 0.5));
  vec3 warm = vec3(0.94, 0.91, 0.86); // parchment
  vec3 glow = vec3(0.98, 0.95, 0.88); // candlelight
  vec3 color = mix(warm, glow, noise * 0.08);
  gl_FragColor = vec4(color, 1.0);
}
```

**Guardrails:**
- Deferred to Phase 4e — only after tools ship and the site is stable.
- Isolated to hero only — never on article pages.
- If it feels gimmicky or causes performance issues, revert to CSS grain overlay (already in Phase 0).
- Branch: `feat/webgl-warmth` — separate from main editorial effects.

**Cost:** ~3 KB JS. Only on homepage. Lazy-loaded or inlined.

## Anti-Nausea Guarantee

All motion effects in this plan use **reveal-in-place** mechanics:

| Technique | Motion Type | Nausea Risk |
|---|---|---|
| `clip-path: polygon()` ink spread | Mask boundary animates over stationary content | **Zero** |
| `clip-path: inset()` text-swap | Text is clipped, not translated across viewport | **Zero** |
| `opacity` + `transform: translateY(8px)` char reveal | Subtle vertical fade, no 3D depth | **Zero** |
| CSS `mask` edge-fade | Static mask, no motion | **Zero** |
| `stroke-dashoffset` line draw | SVG path draws in place | **Zero** |
| Canvas 2D progress rings | Static canvas, animated strokes | **Zero** |
| WebGL warmth layer (Phase 4e) | Full-screen color shift, no camera movement | **Near-zero** |

**Banned:** `rotateY()`, `translateX(100vw)`, `perspective` on text blocks, parallax scrolling on body, scroll-jacking libraries. These trigger vestibular mismatch.

## Guardrails & Rollback Strategy

Since "fortune favors the bold," every risky effect has an escape hatch:

| Effect | Risk | Rollback |
|---|---|---|
| **CharReveal.astro** | DOM bloat, gimmicky feel | Delete component. Fall back to existing `inkSettle` blur-reveal on one element. |
| **TextScramble.astro** | Cognitive load on labels | Remove from stamps. Keep only on calculator results (Phase 4c). |
| **WebGL warmth layer (4e)** | Performance, shader compilation failure | Delete `HeroCanvas.astro`. CSS grain overlay (Phase 0) already provides tactile feel. |
| **Ink spread reveal** | Browser support gaps (no Houdini) | Astro View Transitions API is the fallback. CSS `clip-path` without `@property` degrades to simple opacity fade. |
| **Mix-blend-difference nav** | Accessibility contrast issues on certain images | Remove `mix-blend-mode`. Add back a subtle background box with `backdrop-filter: blur(4px)`. |

All effects live in their own commits on `feat/editorial-effects`. Revert any single commit to remove an effect.
