## JS Budget Philosophy

The zero-JS default applies strictly to **article body pages** — readers should read without JavaScript. The **surfaces around the content** (hero, transitions, hub pages, tools) are where editorial personality lives. Those use targeted JS without betraying the notebook aesthetic.

| Surface | JS Budget | Rationale |
|---|---|---|
| Article body (`/insurance/<slug>/`, `/meat/<slug>/`) | **Zero** | Text renders without JS. Binding Thread is the only exception (~500 B inline). |
| Homepage hero + flat list | **Lightweight** | Session-aware skip, scroll-reveal IntersectionObserver, grain overlay. |
| Page transitions (home → article) | **CSS-only** | Ink Spread reveal uses @property + clip-path. Astro View Transitions as fallback. No JS for transitions. |
| Hub pages (`/insurance`, `/meat`, `/archive`) | **Medium** | Pause-on-offscreen observer, CSS `:has()` filters, mix-blend nav. |
| Tools (`/tools/<slug>`) | **React Island** | Canvas charts, interactive calculators, text decode on results. |
| Dedicated "experience" pages (future) | **TBD** | Explicitly deferred. No WebGL in Phase 1–4. |

## Format System (Visual Metadata)

Format determines the article page's visual template + header badge. It is NOT a URL segment.

| Format | Header Badge | Layout | Best For |
|---|---|---|---|
| `forensic` | `CASE FILE` | Myth → Evidence → Mechanism → Verdict structure | Insurance teardowns, deep meat investigations |
| `experiment` | `EXPERIMENT LOG` | Setup → Variables → Observations → Results → Limitations | Structured tests, comparisons |
| `field-note` | `FIELD NOTE` | Minimal, timestamp-forward, single column | Quick observations, tips |
| `longform` | (none — default) | Standard prose, single column | Meat techniques, personal essays, general notes |

All four formats share the same MDX component library (Highlight, MarginNote, ScrapCard, CorrectionBlock, VerdictSeal). The format controls the page structure wrapping around the MDX body.

### Step 8 — Visual Foundation

This step installs the CSS-first effects base that all subsequent phases build on. It is the infrastructure layer — not decoration.

#### 8a. Font Metric Fallbacks

Thai fonts have extreme vertical metric differences from system fallbacks. Without `size-adjust` / `ascent-override` / `descent-override`, the first paint shows content at wrong positions, then reflows when web fonts load — visible CLS (Cumulative Layout Shift). Thorgal does this for "Artifex Hand CF" with precise percentage overrides. We do it for all four font families.

Add fallback `@font-face` declarations in `src/styles/global.css`:

```css
@font-face {
  font-family: "Anuphan-fallback";
  src: local("Noto Sans Thai");
  size-adjust: 103.2%;
  ascent-override: 95%;
  descent-override: 25%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "Sarabun-fallback";
  src: local("Noto Sans Thai");
  size-adjust: 106.1%;
  ascent-override: 92%;
  descent-override: 28%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "K2D-fallback";
  src: local("Tahoma");
  size-adjust: 104.5%;
  ascent-override: 88%;
  descent-override: 32%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "IBM Plex Mono-fallback";
  src: local("Courier New");
  size-adjust: 97.8%;
  ascent-override: 96%;
  descent-override: 22%;
  line-gap-override: 0%;
}
```

Update Tailwind `@theme` / `fontFamily` tokens in `global.css`:
```css
--font-display: "Anuphan", "Anuphan-fallback", sans-serif;
--font-thai: "Sarabun", "Sarabun-fallback", sans-serif;
--font-hand: "K2D", "K2D-fallback", sans-serif;
--font-mono: "IBM Plex Mono", "IBM Plex Mono-fallback", monospace;
```

**Note:** Exact `size-adjust` / `ascent-override` / `descent-override` values must be calibrated using [Font Style Matcher](https://s3.amazonaws.com/ftlabs-font-style-matcher/index.html) or equivalent, comparing each web font against its chosen local fallback. The values above are estimates — measure real CLS reduction using Lighthouse or `web-vitals` before/after.

#### 8b. Noise Grain Overlay

Generate a 100x100px grayscale noise texture tile as `public/noise.webp` (~2KB). Add in `global.css`:

```css
.nb-page::after {
  content: '';
  position: fixed;
  inset: 0;
  z-index: 9998;
  pointer-events: none;
  opacity: 0.06;
  background: url('/noise.webp') repeat;
  -webkit-background-size: 100px 100px;
  background-size: 100px 100px;
}

@media (prefers-reduced-motion: reduce) {
  .nb-page::after { display: none; }
}
```

Uses simple `opacity: 0.06` instead of `mix-blend-mode: multiply`. On cream (`#F0EADC`), multiply darkens warm tones into muddy brown — simple opacity is safer and achieves the same paper-grain feel.

**Alternative:** If the grain feels too flat, test `mix-blend-mode: overlay` at 4% opacity. Overlay preserves highlights and shadows while adding texture to midtones — closer to real paper grain than multiply without the muddiness.

#### 8c. @property Custom Properties

Register two animatable custom properties at `:root` level. This is the foundational unlock — without it, CSS cannot animate `clip-path` polygon expressions through `calc()`:

```css
@property --ink-spread {
  syntax: '<percentage>';
  inherits: true;
  initial-value: 100%;
}

@property --paper-offset {
  syntax: '<percentage>';
  inherits: true;
  initial-value: 0%;
}
```

**Browser support:** Chrome 85+, Edge 85+, Safari 15.4+, Firefox 128+. Firefox < 128 falls back to content rendering normally (no entrance animation — acceptable progressive enhancement).

#### 8e. Clip-Path Wipe Keyframes

Reusable utility keyframes. Used by badges, stamps, and section headings in Phase 1+. **Not** replacing `inkSettle` on entries — the blur-to-sharp reveal is more "ink drying on paper" than a hard vertical wipe. These wipes are for stamps and badges only.

```css
@keyframes nb-wipe-up {
  from { clip-path: inset(0 0 100% 0); }
  to   { clip-path: inset(0 0 0 0); }
}

@keyframes nb-wipe-down {
  from { clip-path: inset(100% 0 0 0); }
  to   { clip-path: inset(0 0 0 0); }
}

@keyframes nb-wipe-right {
  from { clip-path: inset(0 0 0 100%); }
  to   { clip-path: inset(0 0 0 0); }
}

@keyframes nb-stamp-reveal {
  from { clip-path: inset(0 100% 0 0); }
  to   { clip-path: inset(0 0 0 0); }
}

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

### 2d. Ink Spread Page Reveal (GLM)

**The flagship entrance effect.** When a reader navigates to any article page for the first time in a session, content reveals through an asymmetric ink-spread polygon animation. On repeat navigation, the animation is skipped.

**Design:** The polygon starts from a center point (ink drop on paper) and expands outward with deliberate asymmetry — wider horizontally (0.52-0.55) than vertically (0.45-0.50) to suggest capillary action through paper fibers. `ease-out` timing means fast initial spread (ink soaks quickly) that decelerates at edges (paper saturates).

**The polygon coefficients create slight asymmetry.** These values should be visually tuned during implementation. The general principle: wider horizontal spread than vertical.

```css
@keyframes ink-reveal {
  from {
    --ink-spread: 100%;
  }
  to {
    --ink-spread: 0%;
  }
}

.nb-ink-reveal {
  clip-path: polygon(
    calc(50% - var(--ink-spread) * 0.52)  calc(50% - var(--ink-spread) * 0.48),
    calc(50% + var(--ink-spread) * 0.48)  calc(50% - var(--ink-spread) * 0.45),
    calc(50% + var(--ink-spread) * 0.55)  calc(50% - var(--ink-spread) * 0.10),
    calc(50% + var(--ink-spread) * 0.50)  calc(50% + var(--ink-spread) * 0.50),
    calc(50% + var(--ink-spread) * 0.42)  calc(50% + var(--ink-spread) * 0.55),
    calc(50% - var(--ink-spread) * 0.45)  calc(50% + var(--ink-spread) * 0.52),
    calc(50% - var(--ink-spread) * 0.50)  calc(50% + var(--ink-spread) * 0.35),
    calc(50% - var(--ink-spread) * 0.53)  calc(50% - var(--ink-spread) * 0.05)
  );
  animation: ink-reveal 0.8s ease-out forwards;
  will-change: clip-path;
}

/* Progressive enhancement: skip for browsers without @property */
@supports not (at-rule: @property) {
  .nb-ink-reveal {
    clip-path: none;
    animation: none;
  }
}
```

**Astro View Transitions fallback (Kimi):** For cross-page navigation transitions in browsers that support View Transitions API but not `@property`:

```css
::view-transition-new(root) {
  animation: ink-reveal 0.5s ease-out forwards;
}
::view-transition-old(root) {
  animation: none;
  opacity: 0;
}
```

**JavaScript in `ArticleLayout.astro` (~200 bytes inline):**

```js
(function() {
  var K = 'bf_ink';
  var m = document.querySelector('main');
  if (!m || sessionStorage.getItem(K)) {
    if (m) m.style.clipPath = 'none';
    return;
  }
  m.classList.add('nb-ink-reveal');
  m.addEventListener('animationend', function() {
    m.style.clipPath = 'none';
    m.classList.remove('nb-ink-reveal');
    sessionStorage.setItem(K, '1');
  }, { once: true });
  setTimeout(function() {
    if (m.classList.contains('nb-ink-reveal')) {
      m.style.clipPath = 'none';
      m.classList.remove('nb-ink-reveal');
      sessionStorage.setItem(K, '1');
    }
  }, 2000);
})();
```

**Anti-nausea guarantee:** Content is already in the DOM and stationary. Only the visibility boundary (clip-path mask) animates. Zero DOM movement, zero layout shift, zero nausea. Session-aware — fires once, then content appears instantly on subsequent navigations.

## Effects Research Reference

For future reference, the primary techniques stolen and their sources:

### From Thorgal (thorgal.com)

Astro v6.1.8, 12 Web Components, Taxi page transitions, WebGL 3D doors.

1. **Metric-compatible font fallback** — `size-adjust` / `ascent-override` / `descent-override` on `@font-face` fallback families to eliminate CLS. Thorgal does this for "Artifex Hand CF" with precise percentage overrides against Arial.
2. **Character-by-character text reveal** — `<c-text-reveal>` with `data-stagger="0.01"`. **Rejected for beef.im** — DOM bloat concern.
3. **clip-path inset wipe-up badges** — `clip-path: inset(0 0 100% 0)` → `inset(0 0 0 0)` for stamp reveals.
4. **mix-blend-difference navigation** — Fixed sidebar dots and scroll indicator auto-contrast against 3D backgrounds.
5. **Vertical section snap via CSS custom properties** — `--door-index` drives `translate-y-[calc()]` for fullscreen sections without scroll-snap library.

### From Pixila (pixila.net)

Astro v5.17.3, CSS-first approach, no WebGL at all, Partytown for analytics.

1. **@property + clip-path polygon page reveal** — The flagship technique. `CSS.registerProperty({ name: "--offset", syntax: "<percentage>" })` enables CSS to animate custom properties through `calc()` inside `clip-path: polygon()`. The page reveals through a 10-point asymmetric polygon.
2. **Dual-layer clip-path text hover** — Two stacked `<span>` with opposing `clip-path: inset()` + `skewY(2deg)`. Mechanical/instrument text swap on hover. 350ms timing.
3. **SVG stroke-dashoffset annotation pointers** — Staggered diamond → line (0.8s `stroke-dashoffset` draw) → text (0.25s fade). Pixila's `model-ui` animation system.
4. **Noise texture overlay** — Full-page `.webp` at 8% opacity with `mask: linear-gradient` edge fading and `mix-blend-mode: multiply`. beef.im uses 6% simple opacity (multiply muddies on cream).
5. **Ghost text echo layer** — Text rendered twice at full + 15% opacity, stacked via CSS Grid. beef.im adapts to 8% + `translateY(4px)` for ink-bleed-through-paper.
6. **Session-aware preloader** — `sessionStorage` skips animations on repeat visits. Applied to both hero skip and ink reveal.
7. **CSS-only marquee with viewport pausing** — Not used (no marquee content), but `animation-play-state: paused` technique noted for future.
8. **Animated link underlines** — `::after` pseudo-element with `width: 0` → `width: 100%` on hover. Pixila uses this on footer links.
