## JS Budget Philosophy

The zero-JS default applies strictly to **article body pages**. The **surfaces around the content** (hero, transitions, hub pages, tools) are where editorial personality lives.

| Surface | JS Budget | Rationale |
|---|---|---|
| Article body (`/insurance/<slug>/`, etc.) | **Zero** | Text renders without JS. Binding Thread is the only exception (~500B inline). |
| Homepage hero + flat list | **Lightweight** | Session-aware skip, scroll-reveal IntersectionObserver, grain overlay. |
| Page transitions | **CSS-only** | Clip-path reveals are pure CSS. No JS for navigation transitions. |
| Hub pages (`/insurance`, `/meat`, `/archive`) | **Medium** | Pause-on-offscreen observer, marquees, CSS `:has()` filters. |
| Tools (`/tools/<slug>`) | **React Island** | Canvas charts, interactive calculators — already in plan. |
| Dedicated "experience" pages (future) | **TBD** | Explicitly deferred. No WebGL in Phase 1–4. |

## Format System (Visual Metadata)

Format determines the article page's visual template + header badge. It is NOT a URL segment.

| Format | Header Badge | Layout | Best For |
|---|---|---|---|
| `forensic` | `CASE FILE` | Myth → Evidence → Mechanism → Verdict structure | Insurance teardowns, deep meat investigations |
| `experiment` | `EXPERIMENT LOG` | Setup → Variables → Observations → Results → Limitations | Structured tests, comparisons |
| `field-note` | `FIELD NOTE` | Minimal, timestamp-forward, single column | Quick observations, tips |
| `longform` | (none — default) | Standard prose, single column | Meat techniques, personal essays, general notes |

All four formats share the same MDX component library (Highlight, MarginNote, ScrapCard, CorrectionBlock, VerdictSeal). The format controls the page structure wrapping around the MDX body.

**`[...slug].astro` conditional rendering:**

```astro

const format = entry.data.format || 'longform';
const watermark = collection === 'insurance' ? 'analysis'
  : collection === 'meat' ? 'beef'
  : 'note';

<!-- Header badge based on format -->
{format === 'forensic' && <span class="nb-stamp">CASE FILE</span>}
{format === 'experiment' && <span class="nb-stamp">EXPERIMENT LOG</span>}
{format === 'field-note' && <span class="nb-stamp">FIELD NOTE</span>}

<!-- Watermark derived from collection -->
<Footer type={watermark} />
```

**Step 8 — Font Metric Fallbacks (NEW — from GLM-5.1 via Thorgal):**

Thai fonts (Sarabun, Anuphan, K2D) have significant vertical metric divergence from their system fallbacks. Without `size-adjust`/`ascent-override`/`descent-override` on `@font-face` fallback declarations, the first paint shows content at wrong positions before the web font loads — visible Cumulative Layout Shift (CLS). Thorgal uses this technique for "Artifex Hand CF" to achieve CLS-free font loading.

Add fallback `@font-face` declarations for each font family in `src/styles/global.css`:

```css
@font-face {
  font-family: "Anuphan-fallback";
  src: local("Noto Sans Thai");
  size-adjust: 103.2%;
  ascent-override: 95%;
  descent-override: 25%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "Sarabun-fallback";
  src: local("Noto Sans Thai");
  size-adjust: 106.1%;
  ascent-override: 92%;
  descent-override: 28%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "K2D-fallback";
  src: local("Tahoma");
  size-adjust: 104.5%;
  ascent-override: 88%;
  descent-override: 32%;
  line-gap-override: 0%;
}

@font-face {
  font-family: "IBM Plex Mono-fallback";
  src: local("Courier New");
  size-adjust: 97.8%;
  ascent-override: 96%;
  descent-override: 22%;
  line-gap-override: 0%;
}
```

Update Tailwind `@theme`/`fontFamily` tokens to use fallback chains:
```css
--font-display: "Anuphan", "Anuphan-fallback", sans-serif;
--font-thai: "Sarabun", "Sarabun-fallback", sans-serif;
--font-hand: "K2D", "K2D-fallback", sans-serif;
--font-mono: "IBM Plex Mono", "IBM Plex Mono-fallback", monospace;
```

**Note:** Exact `size-adjust`/`ascent-override`/`descent-override` values must be calibrated using [Font Style Matcher](https://s3.amazonaws.com/ftlabs-font-style-matcher/index.html) comparing each web font against its chosen local fallback. Values above are estimates — measure CLS reduction with Lighthouse before/after.

### 1b-2. SVG Stroke-Dashoffset Rule Draw

Replace the existing `ruleDraw` keyframe (`scaleX(0)` → `scaleX(1)`) with an SVG `<line>` animated via `stroke-dashoffset`. Same timing, same placement. "Pen drawing a line" instead of "bar sliding out." Stolen from Pixila's `model-ui-stroke` animation.

**Current:** CSS `@keyframes ruleDraw { from { transform: scaleX(0); } to { transform: scaleX(1); } }` applied to a `<div>`.

**Replacement:** An inline SVG `<line>` with `stroke-dasharray="100"` and `stroke-dashoffset="100"` → `0` transition.

```css
.nb-rule-line {
  stroke-dasharray: 100;
  stroke-dashoffset: 100;
  animation: ruleDraw 900ms cubic-bezier(0.4, 0, 0.2, 1) forwards;
}
@keyframes ruleDraw {
  to { stroke-dashoffset: 0; }
}
```

### 1b-3. Session-Aware Hero Skip

Returning readers shouldn't endure the 4.8s hero timeline every visit. A `sessionStorage` flag (reset per session, persists across page loads) skips the hero animation on second+ visit within a session. Stolen from Pixila's `pixila_preloaded` preloader pattern.

**Implementation:** ~5 lines of vanilla JS inline in `index.astro`.

```js
// In index.astro <script> — runs before hero animation starts
if (sessionStorage.getItem('bf-hero-seen')) {
  document.querySelector('.hero').classList.add('hero--instant');
} else {
  sessionStorage.setItem('bf-hero-seen', '1');
}
```

```css
/* When .hero--instant is present, all hero elements start at their final state */
.hero--instant .hero-watermark,
.hero--instant .hero-burst,
.hero--instant .hero-rule,
.hero--instant .hero-eyebrow,
.hero--instant .hero-title,
.hero--instant .hero-subhead,
.hero--instant .hero-cta,

  opacity: 1 !important;
}
```

### 2d-1. Ink Spread Page Reveal (from GLM-5.1 via Pixila)

The flagship article entrance effect. When a reader navigates to any article page for the first time in a session, the content reveals through an asymmetric ink-spread polygon animation — like a drop of ink hitting paper and seeping into the fibers. On repeat navigation within the same session, the animation is skipped.

**Design:** The polygon starts from a central convergence point and expands outward asymmetrically — wider horizontally than vertically, with deliberate coefficient offsets (0.52, 0.48, 0.55, 0.45...) to simulate capillary action through paper fibers. The metaphor is on-brand: forensic journal, ink on paper.

**Pre-requisite:** Registers `--ink-spread` via `@property` in Phase 0 Step 8 (already built into the plan). This enables CSS to animate `clip-path: polygon()` expressions through `calc()`.

**CSS in `global.css`:**

```css
@property --ink-spread {
  syntax: '<percentage>';
  inherits: true;
  initial-value: 100%;
}

@keyframes ink-reveal {
  from { --ink-spread: 100%; }
  to   { --ink-spread: 0%; }
}

.nb-ink-reveal {
  clip-path: polygon(
    /* Vertices converge at center when --ink-spread = 100% */
    /* Spread to full rectangle when --ink-spread = 0% */
    calc(50% - var(--ink-spread) * 0.52)  calc(50% - var(--ink-spread) * 0.48),
    calc(50% + var(--ink-spread) * 0.48)  calc(50% - var(--ink-spread) * 0.45),
    calc(50% + var(--ink-spread) * 0.55)  calc(50% - var(--ink-spread) * 0.10),
    calc(50% + var(--ink-spread) * 0.50)  calc(50% + var(--ink-spread) * 0.50),
    calc(50% + var(--ink-spread) * 0.42)  calc(50% + var(--ink-spread) * 0.55),
    calc(50% - var(--ink-spread) * 0.45)  calc(50% + var(--ink-spread) * 0.52),
    calc(50% - var(--ink-spread) * 0.50)  calc(50% + var(--ink-spread) * 0.35),
    calc(50% - var(--ink-spread) * 0.53)  calc(50% - var(--ink-spread) * 0.05)
  );
  animation: ink-reveal 0.8s ease-out forwards;
  will-change: clip-path;
}

/* Progressive enhancement: skip for browsers without @property support */
@supports not (at-rule: @property) {
  .nb-ink-reveal {
    clip-path: none;
    animation: none;
  }
}
```

**Timing:** 0.8s with `ease-out` — initial expansion is fast (ink soaks quickly), decelerates as it reaches the edges (ink saturates paper).

**JavaScript in `ArticleLayout.astro` (inline, ~200B):**

```js
(function() {
  var K = 'beef_ink';
  var m = document.querySelector('main');
  if (!m || sessionStorage.getItem(K)) {
    if (m) m.style.clipPath = 'none';
    return;
  }
  m.classList.add('nb-ink-reveal');
  m.addEventListener('animationend', function() {
    m.style.clipPath = 'none';
    m.classList.remove('nb-ink-reveal');
    sessionStorage.setItem(K, '1');
  }, { once: true });
  // Safety fallback: skip after 2s if animation fails
  setTimeout(function() {
    if (m.classList.contains('nb-ink-reveal')) {
      m.style.clipPath = 'none';
      m.classList.remove('nb-ink-reveal');
      sessionStorage.setItem(K, '1');
    }
  }, 2000);
})();
```

**Browser support:** Chrome 85+, Edge 85+, Safari 15.4+, Firefox 128+. Firefox < 128 falls back to content rendering normally (no entrance animation — acceptable progressive enhancement).

**The polygon coefficients** create slight asymmetry — the "ink" doesn't spread in a perfect circle. These values should be visually tuned during implementation. The principle: wider horizontal spread (0.52–0.55) than vertical (0.45–0.50) to suggest capillary action through paper fibers.

### 2d-5. Pause-on-Offscreen Animations

Auto-playing animations (marquees, scroll-reveal cascade) pause when the element leaves the viewport. Resumes when re-entering. Performance hygiene, not decoration. Stolen from Pixila's viewport-aware marquee pattern. Caught by Kimi K2.6.

**Tech:** `IntersectionObserver` (~300B inline JS).

```js
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    entry.target.style.animationPlayState = entry.isIntersecting ? 'running' : 'paused';
  });
}, { threshold: 0.1 });
document.querySelectorAll('.nb-marquee, .nb-animate').forEach(el => observer.observe(el));
```

**Apply to:** Any element with `.nb-marquee` or `.nb-animate` class on hub and archive pages.

## Phase 4c — Calculator Canvas Effects

> **Source:** Thorgal.com loader canvas + Pixila.net (suspected) text decode pattern
> **Budget:** ~3KB vanilla JS total. No Three.js, no WebGL, no React Three Fiber.

### 4c-1. 2D Canvas Progress Charts

A lightweight `<canvas>` element with 2D rendering context for COI cost projection charts, IRR comparison graphs, and premium breakdown visualizations. Same technique Thorgal uses for its loader ring — pure Canvas 2D API, no WebGL context, no shader compilation overhead.

**Why not SVG/Chart.js:** Chart libraries add 50–100KB+. A hand-rolled `<canvas>` chart at 1–2KB fits the "zero bloat" doctrine and matches the hand-drawn notebook aesthetic — we can render graph lines that look like they were drawn with a fountain pen.

**Implementation:** A ~2KB vanilla JS module `src/components/tools/ChartCanvas.ts` that accepts a data array + config and renders to a `<canvas>` element. Used inside React tool islands (hydrated with `client:load`).

```ts
// Minimal API surface
interface ChartConfig {
  data: number[];
  width: number;
  height: number;
  lineColor: string; // var(--color-red) or var(--color-navy)
  drawStyle: 'sketch' | 'precise';
}
function drawChart(canvas: HTMLCanvasElement, config: ChartConfig): void;
```

The `'sketch'` style adds slight jitter to line points for a hand-drawn feel.

Creates a "forensic unearthing" moment where the truth mechanically resolves.

Stolen from Pixila's suspected text-scramble class (confirmed by Gemini 3.1 Pro and Kimi K2.6 analyses — not personally verified in source, but the concept is mechanically sound and independently reproducible).

```js
class TextDecode {

  }
}
```

Apply to: COI cost totals, IRR percentage results, premium comparison deltas. Only on calculator results — never on editorial body text.

## Anti-Nausea Guarantee

All motion effects in this plan use **reveal-in-place** mechanics. Content is already loaded and stationary — only the mask or opacity boundary animates across it. No DOM elements translate across the viewport. This avoids the vestibular mismatch that causes motion sickness with page-flip or 3D transform approaches.

| Technique | Motion Type | Nausea Risk |
|---|---|---|
| `clip-path: inset()` page reveal | Mask boundary unrolls over stationary content | **Zero** |
| `clip-path: inset()` text-swap | Text is clipped, not translated across viewport | **Zero** |
| `opacity` + `filter: blur()` inkSettle | Subtle blur-to-sharp reveal, no translation | **Zero** |
| CSS `mask` edge-fade | Static mask, no motion | **Zero** |
| `stroke-dashoffset` line draw | SVG path draws in place | **Zero** |
| Canvas 2D progress rings | Static canvas, animated strokes | **Zero** |
| `mix-blend-mode: difference` nav | Color inversion on scroll, no motion | **Zero** |
| `clip-path: polygon()` ink-spread reveal | Mask boundary expands from center over stationary content | **Zero** |
| SVG annotation pointers (diamond→line→text) | SVG elements draw in place, no translation | **Zero** |

**Explicitly banned:** `rotateY()`, `rotateX()` on text blocks, `translateX(100vw)` full-page slides, `perspective()` on body content, parallax scrolling on text columns, scroll-jacking libraries (Lenis, Locomotive Scroll, smooth-scroll.js). These trigger vestibular mismatch — your eyes track motion your inner ear doesn't feel.
