## Philosophy: "Zero JS on Content, Editorial Moments Get the Juice"

The original plan's zero-JS default applies strictly to **article body pages** — readers should read without JavaScript. The **surfaces around the content** (hero, transitions, hub pages, tools) are where editorial personality lives. Those use targeted JS/WebGL without betraying the notebook aesthetic.

| Surface | JS Budget | Rationale |
|---|---|---|
| Article body (`/insurance/<slug>/`, `/meat/<slug>/`) | **Zero** | Text renders without JS. Binding Thread is the only exception (~500 B inline). |
| Homepage hero + flat list | **Lightweight** | Session-aware skip, text decode, scroll-reveal, grain overlay. |
| Page transitions (home → article, article → article) | **Lightweight** | CSS clip-path reveals + Astro View Transitions API. Content stays anchored, mask animates. |
| Hub pages (`/insurance`, `/meat`, `/archive`) | **Medium** | Horizontal scrolls, animated filters, marquees, pause-on-offscreen. |
| Tools (`/tools/<slug>`) | **React Island** | Canvas charts, interactive calculators — already in plan. |
| Dedicated "experience" pages (future) | **Full** | WebGL shaders, 3D — isolated to specific moments. |

## Format System (Visual Metadata)

Format determines the article page's visual template + header badge. It is NOT a URL segment.

| Format | Header Badge | Layout | Best For |
|---|---|---|---|
| `forensic` | `CASE FILE` | Myth → Evidence → Mechanism → Verdict structure | Insurance teardowns, deep meat investigations |
| `experiment` | `EXPERIMENT LOG` | Setup → Variables → Observations → Results → Limitations | Structured tests, comparisons |
| `field-note` | `FIELD NOTE` | Minimal, timestamp-forward, single column | Quick observations, tips |
| `longform` | (none — default) | Standard prose, single column | Meat techniques, personal essays, general notes |

All four formats share the same MDX component library (Highlight, MarginNote, ScrapCard, CorrectionBlock, VerdictSeal). The format controls the page structure wrapping around the MDX body.

**`[...slug].astro` conditional rendering:**

```astro
---
const collection = entry.collection; // 'insurance' | 'meat' | 'note'
const format = entry.data.format || 'longform';
const watermark = collection === 'insurance' ? 'analysis'
  : collection === 'meat' ? 'beef'
  : 'note';
---
<!-- Header badge based on format -->
{format === 'forensic' && <span class="nb-stamp">CASE FILE</span>}
{format === 'experiment' && <span class="nb-stamp">EXPERIMENT LOG</span>}
{format === 'field-note' && <span class="nb-stamp">FIELD NOTE</span>}

<!-- Watermark derived from collection -->
<Footer type={watermark} />
```

### 1g. Editorial Surfaces — NEW

This section unlocks the stealable techniques from Thorgal.com and Pixila.net for Phase 1. Article body remains zero-JS. These effects apply to hero, navigation, CTAs, and global surfaces.

#### 1g-1. Global Noise/Grain Overlay (Pixila)

**What:** A fixed full-screen pseudo-element with a repeating noise texture at 5–8% opacity. Makes digital parchment feel like physical paper.

**Where:** `BaseLayout.astro` — added as a single `<div>` or `::after` pseudo-element on `<body>`.

**Tech:** CSS only. `pointer-events: none`, `position: fixed`, `inset: 0`, `z-index: 9999`, `opacity: 0.06`, `background-image: url(/noise.webp)`, `background-repeat: repeat`, `background-size: 128px`.

**Asset:** Generate a 128×128px seamless noise tile (neutral grayscale, subtle paper fiber). Save as `public/noise.webp` (~2 KB).

**CSS in `global.css`:**
```css
body::after {
  content: '';
  position: fixed;
  inset: 0;
  z-index: 9999;
  pointer-events: none;
  opacity: 0.06;
  background-image: url('/noise.webp');
  background-repeat: repeat;
  background-size: 128px;
}
@media (prefers-reduced-motion: reduce) {
  body::after { display: none; }
}
```

**Cost:** 0 B JS. ~2 KB asset. One-time addition.

#### 1g-2. Session-Aware Hero Skip (Pixila)

**What:** On first visit, the hero plays its full timeline (typewriter + inkSettle). On return visit (same session), the hero appears instantly — no animation delay.

**Where:** `index.astro` hero section. `Hero.astro` reads a `sessionStorage` flag.

**Tech:** Vanilla JS inline script (~150 B). Sets `sessionStorage.setItem('beef_hero_seen', '1')` after first load. On subsequent loads, adds `.is-returning` class to hero wrapper, which overrides animation durations to 0s.

**CSS:**
```css
.hero.is-returning .hero-eyebrow span,
.hero.is-returning .hero-title,
.hero.is-returning .hero-lede {
  animation-duration: 0s !important;
  opacity: 1 !important;
  filter: none !important;
  transform: none !important;
}
```

**Cost:** ~150 B inline JS. Zero layout shift.

#### 1g-3. Hover Text-Swap on CTAs (Pixila)

**What:** On hover, CTA text doesn't just change color — the visible text is clipped away while secondary text pushes up from below. Tactile, mechanical, editorial.

**Where:** "อ่านต่อ" buttons on `NotebookEntry.astro`, footer links, nav links.

**Tech:** CSS only. Two stacked `<span>` elements inside a button. Parent has `overflow: hidden`, `position: relative`. On hover: first span `translateY(-100%)` + `clip-path: inset(0 0 100% 0)`, second span `translateY(0)` from `translateY(100%)`.

**CSS:**
```css
.nb-swap-btn {
  position: relative;
  overflow: hidden;
  display: inline-block;
}
.nb-swap-btn span {
  display: block;
  transition: transform 0.3s cubic-bezier(0.2, 1, 0.2, 1),
              clip-path 0.3s cubic-bezier(0.2, 1, 0.2, 1);
}
.nb-swap-btn .swap-out {
  clip-path: inset(0 0 0 0);
}
.nb-swap-btn:hover .swap-out {
  transform: translateY(-100%);
  clip-path: inset(0 0 100% 0);
}
.nb-swap-btn .swap-in {
  position: absolute;
  top: 0; left: 0;
  transform: translateY(100%);
  clip-path: inset(100% 0 0 0);
}
.nb-swap-btn:hover .swap-in {
  transform: translateY(0);
  clip-path: inset(0 0 0 0);
}
```

**Cost:** 0 B JS.

#### 1g-4. Animated Link Underlines (Pixila)

**What:** Inline links grow an underline from `width: 0` to `width: 100%` on hover, using a pseudo-element.

**Where:** Footer site-map row, inline article links, static page prose links.

**Tech:** CSS only. `::after` pseudo-element with `width: 0` → `width: 100%` transition.

**CSS:**
```css
.nb-link-underline {
  position: relative;
  text-decoration: none;
}
.nb-link-underline::after {
  content: '';
  position: absolute;
  bottom: -1px; left: 0;
  width: 0; height: 1px;
  background: currentColor;
  transition: width 0.3s ease;
}
.nb-link-underline:hover::after {
  width: 100%;
}
```

**Cost:** 0 B JS.

#### 1g-5. Corner Frame Marks (Pixila)

**What:** Small `+` or corner bracket marks at the corners of article header images, pull quotes, or scrap cards. Reinforces the "draft/blueprint" notebook aesthetic.

**Where:** `ArticleLayout.astro` header area, `<ScrapCard>` component, pull quote blocks.

**Tech:** CSS pseudo-elements. `::before` and `::after` on a wrapper. Four corners positioned absolutely.

**CSS:**
```css
.nb-corner-frame {
  position: relative;
}
.nb-corner-frame::before,
.nb-corner-frame::after {
  content: '+';
  position: absolute;
  font-family: var(--font-mono);
  font-size: 10px;
  color: var(--color-gold);
  opacity: 0.5;
}
.nb-corner-frame::before { top: -4px; left: -4px; }
.nb-corner-frame::after { top: -4px; right: -4px; }
.nb-corner-frame .corner-bl::before,
.nb-corner-frame .corner-br::after {
  content: '+';
  position: absolute;
  bottom: -4px;
}
```

**Cost:** 0 B JS.

#### 1g-6. SVG Stroke-Dashoffset on Rules (Thorgal + Pixila)

**What:** Red rules and section dividers draw themselves onto the page via SVG `stroke-dashoffset` animation, rather than scaling from `width: 0`.

**Where:** Hero rule (`Hero.astro`), section dividers in static pages, `VerdictSeal.astro` circle.

**Tech:** CSS `@keyframes`. Replace current `ruleDraw` (scaleX) with `ruleStrokeDraw` (stroke-dashoffset) for SVG `<line>` elements.

**CSS:**
```css
@keyframes ruleStrokeDraw {
  from { stroke-dashoffset: 120; }
  to   { stroke-dashoffset: 0; }
}
.hero-rule-svg line {
  stroke: var(--color-red);
  stroke-width: 2;
  stroke-dasharray: 120;
  stroke-dashoffset: 120;
  animation: ruleStrokeDraw 0.8s cubic-bezier(0.2, 1, 0.2, 1) forwards;
}
```

**Note:** Keep existing `ruleDraw` for CSS `div` rules. Add `ruleStrokeDraw` for SVG rules. Both can coexist.

**Cost:** 0 B JS.

### 2d. Motion Surfaces — NEW

#### 2d-1. Clip-Path Polygon Page Reveal (Pixila)

**What:** When navigating homepage → article (or article → article), the new page doesn't hard-swap. Instead, a geometric clip-path mask animates to reveal the new content in place. Content stays anchored — only the visibility boundary changes. Zero nausea.

**Where:** All route transitions. Uses Astro's native View Transitions API (`<ViewTransitions />` in `BaseLayout.astro`) with a custom CSS clip-path animation.

**Tech:** CSS `@keyframes` + `clip-path: polygon()` + Astro `swap()` transition. The mask starts as a narrow vertical slit and expands horizontally.

**CSS:**
```css
@keyframes pageReveal {
  from {
    clip-path: polygon(50% 0%, 50% 0%, 50% 100%, 50% 100%);
  }
  to {
    clip-path: polygon(0% 0%, 100% 0%, 100% 100%, 0% 100%);
  }
}
::view-transition-new(root) {
  animation: pageReveal 0.5s cubic-bezier(0.2, 1, 0.2, 1) forwards;
}
::view-transition-old(root) {
  animation: pageReveal 0.3s cubic-bezier(0.2, 1, 0.2, 1) reverse forwards;
}
```

**Note:** Astro View Transitions API requires `astro.config.mjs` to enable: `experimental: { viewTransitions: true }` or use the `<ViewTransitions />` component.

**Cost:** 0 B JS (handled by browser + Astro). ~20 lines CSS.

#### 2d-2. Text Scramble / Decode (Pixila — claimed)

**What:** Category stamps and format badges (e.g., `CASE FILE`, `EXPERIMENT LOG`) initially appear as scrambled random characters (`!<>-_/[]{}—=+*^?#________`), then mechanically decode to the final text. Like a teletype or declassified document.

**Where:** Homepage format badges, hub page titles, article header stamps.

**Tech:** Vanilla JS class (~1 KB). Uses `requestAnimationFrame` to cycle characters before settling. Triggered on `IntersectionObserver` (appears when scrolled into view) or page load.

**Implementation:** `src/components/TextScramble.astro` — reusable Astro component.

```js
// ~1 KB vanilla JS
class TextScramble {
  constructor(el) {
    this.el = el;
    this.chars = '!<>-_/[]{}—=+*^?#________';
    this.original = el.textContent;
  }
  decode() {
    const length = this.original.length;
    let iter = 0;
    const tick = () => {
      this.el.textContent = this.original
        .split('')
        .map((char, i) => {
          if (i < iter) return this.original[i];
          return this.chars[Math.floor(Math.random() * this.chars.length)];
        })
        .join('');
      if (iter < length) {
        iter += 1 / 3;
        requestAnimationFrame(tick);
      }
    };
    tick();
  }
}
```

**Cost:** ~1 KB JS. Only loaded on pages that use it (homepage, hubs).

### 4e. WebGL Warmth Layer (Pixila — claimed, future)

**What:** A single full-screen fragment shader on the hero background only. Slow warm candlelight flicker over parchment, or subtle paper fiber texture. Not a video, not a CSS gradient — a living, breathing light layer.

**Where:** Homepage hero background only. Behind the notebook grid, above the parchment base.

**Tech:** Raw WebGL (not Three.js). Single `<canvas>` element with a fragment shader. ~3 KB.

**Why raw WebGL, not Three.js:** Three.js adds ~150 KB. A single fullscreen quad with a fragment shader is ~50 lines of WebGL boilerplate + the shader itself. No library needed.

**Shader concept (GLSL):**
```glsl
// Simplex noise-based warm flicker
uniform float u_time;
uniform vec2 u_resolution;

void main() {
  vec2 uv = gl_FragCoord.xy / u_resolution;
  float noise = snoise(vec3(uv * 3.0, u_time * 0.5));
  vec3 warm = vec3(0.94, 0.91, 0.86); // parchment
  vec3 glow = vec3(0.98, 0.95, 0.88); // candlelight
  vec3 color = mix(warm, glow, noise * 0.08);
  gl_FragColor = vec4(color, 1.0);
}
```

**Cost:** ~3 KB JS. Only on homepage. Lazy-loaded or inlined.

**Note:** This is Phase 4e — explicitly deferred until after tools ship. It is the only WebGL in the entire site.

## Anti-Nausea Guarantee

All motion effects in this plan use **reveal-in-place** mechanics:

| Technique | Motion Type | Nausea Risk |
|---|---|---|
