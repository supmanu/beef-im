## Format System (Visual Metadata)

Format determines the article page's visual template + header badge. It is NOT a URL segment.

| Format | Header Badge | Layout | Best For |
|---|---|---|---|
| `forensic` | `CASE FILE` | Myth → Evidence → Mechanism → Verdict structure | Insurance teardowns, deep meat investigations |
| `experiment` | `EXPERIMENT LOG` | Setup → Variables → Observations → Results → Limitations | Structured tests, comparisons |
| `field-note` | `FIELD NOTE` | Minimal, timestamp-forward, single column | Quick observations, tips |
| `longform` | (none — default) | Standard prose, single column | Meat techniques, personal essays, general notes |

All four formats share the same MDX component library (Highlight, MarginNote, ScrapCard, CorrectionBlock, VerdictSeal). The format controls the page structure wrapping around the MDX body.

**`[...slug].astro` conditional rendering:**

```astro
---
const collection = entry.collection; // 'insurance' | 'meat' | 'note'
const format = entry.data.format || 'longform';
const watermark = collection === 'insurance' ? 'analysis'
  : collection === 'meat' ? 'beef'
  : 'note';
---
<!-- Header badge based on format -->
{format === 'forensic' && <span class="nb-stamp">CASE FILE</span>}
{format === 'experiment' && <span class="nb-stamp">EXPERIMENT LOG</span>}
{format === 'field-note' && <span class="nb-stamp">FIELD NOTE</span>}

<!-- Watermark derived from collection -->
<Footer type={watermark} />
```

### 1b-1. Paper Grain Overlay

A fixed full-screen noise texture over the notebook parchment. Adds tactile paper feel without touching layout or content flow. Stolen from Pixila's fixed noise overlay (`opacity: 0.08` → we use `0.05` for subtlety).

**Implementation:** One `<div>` in `BaseLayout.astro` above the closing body tag. Three CSS rules in `global.css`.

```html
<div class="nb-grain" aria-hidden="true"></div>
```

```css
.nb-grain {
  position: fixed;
  inset: 0;
  z-index: 9998;
  pointer-events: none;
  opacity: 0.05;
  background-image: url('/noise.webp');
  background-repeat: repeat;
  background-size: 128px 128px;
}
```

**Noise texture:** Generate a 128×128 WebP with 2-color noise at build time or use a static asset. This is the single highest-impact, lowest-risk effect from the source analysis.

### 1b-2. SVG Stroke-Dashoffset Rule Draw

Replace the existing `ruleDraw` keyframe (`scaleX(0)` → `scaleX(1)`) with an SVG `<line>` animated via `stroke-dashoffset`. Same timing, same placement. "Pen drawing a line" instead of "bar sliding out." Stolen from Pixila's `model-ui-stroke` animation.

**Current:** CSS `@keyframes ruleDraw { from { transform: scaleX(0); } to { transform: scaleX(1); } }` applied to a `<div>`.

**Replacement:** An inline SVG `<line>` with `stroke-dasharray="100"` and `stroke-dashoffset="100"` → `0` transition.

```css
.nb-rule-line {
  stroke-dasharray: 100;
  stroke-dashoffset: 100;
  animation: ruleDraw 900ms cubic-bezier(0.4, 0, 0.2, 1) forwards;
}
@keyframes ruleDraw {
  to { stroke-dashoffset: 0; }
}
```

| Action | File |

| MODIFY | `src/styles/global.css` (replace `ruleDraw` keyframe, add `.nb-rule-line`) |
| MODIFY | All components using red rules (Hero, article headers, static pages) |

### 1b-3. Session-Aware Hero Skip

Returning readers shouldn't endure the 4.8s hero timeline every visit. A `sessionStorage` flag (reset per session, persists across page loads) skips the hero animation on second+ visit within a session. Stolen from Pixila's `pixila_preloaded` preloader pattern.

**Implementation:** ~5 lines of vanilla JS inline in `index.astro`.

```js
// In index.astro <script> — runs before hero animation starts
if (sessionStorage.getItem('bf-hero-seen')) {
  document.querySelector('.hero').classList.add('hero--instant');
} else {
  sessionStorage.setItem('bf-hero-seen', '1');
}
```

```css
/* When .hero--instant is present, all hero elements start at their final state */
.hero--instant .hero-watermark,
.hero--instant .hero-burst,
.hero--instant .hero-rule,
.hero--instant .hero-eyebrow,
.hero--instant .hero-title,
.hero--instant .hero-subhead,
.hero--instant .hero-cta,
.hero--instant .hero-sketch {
  animation: none !important;
  opacity: 1 !important;
}
```

```js
(function() {
  var thread = document.querySelector('.binding-thread');
  if (!thread) return;
  var nodes = [];
  ['mark', '.verdict-seal'].forEach(function(sel) {
    document.querySelectorAll(sel).forEach(function(el) {
      var node = document.createElement('div');
      node.className = 'thread-node';
      node.setAttribute('data-label', el.getAttribute('data-label') || el.textContent.slice(0, 20));
      node.addEventListener('click', function() {
        el.scrollIntoView({ behavior: 'smooth', block: 'center' });
      });
      thread.appendChild(node);
      nodes.push({ node: node, el: el });
    });
  });
  function position() {
    var vh = window.innerHeight;
    nodes.forEach(function(n) {
      var rect = n.el.getBoundingClientRect();
      var pct = (rect.top + rect.height / 2) / document.body.scrollHeight;
      n.node.style.top = (pct * vh) + 'px';
      n.node.style.display = (rect.top < vh && rect.bottom > 0) ? '' : 'none';
    });
  }
  window.addEventListener('scroll', position, { passive: true });
  position();
})();
```

### 2d-1. Hover Text-Swap on CTAs

A mechanical text-swap transition for buttons and interactive links. Two stacked `<span>` elements with opposing `clip-path: inset()`. On hover: top text clips out with a slight skew, bottom text clips in from below — like a slot-machine reel or notebook tab snapping. No bounce, no spring physics. Stolen from Pixila's `<astro-button>` implementation.

**Apply to:** "อ่านต่อ" buttons, pillar hub navigation links, archive "เปิดเล่ม" buttons, "เล่มก่อนหน้า | เล่มถัดไป" pagination.

### 2d-2. Clip-Path Polygon Page Reveal

When Binding Thread activates on an article page, the content reveals via a jagged `clip-path: polygon()` expansion — like a notebook page being unfolded. Content is already loaded and stationary; only the mask boundary animates. Zero DOM movement, zero nausea. Stolen from Pixila's preloader → main content transition.

Requires `CSS.registerProperty()` for smooth `--offset` interpolation (Houdini API, Chromium + Safari 15.4+). Falls back gracefully to a simple opacity fade on unsupported browsers.

```css
@property --offset {
  syntax: '<percentage>';
  inherits: true;
  initial-value: 14%;
}

.nb-page-reveal {
  clip-path: polygon(
    0%   calc(71% + var(--offset)),
    41%  calc(20% + var(--offset)),
    50%  calc(35% + var(--offset)),
    70%  calc(1%  + var(--offset)),
    100% calc(47% + var(--offset)),
    100% calc(75% - var(--offset)),
    70%  calc(29% - var(--offset)),
    50%  calc(63% - var(--offset)),
    41%  calc(48% - var(--offset)),
    0%   calc(99% - var(--offset))
  );
  animation: nb-reveal 0.7s ease-in-out forwards;
}
@keyframes nb-reveal {
  to { --offset: -80%; }
}
```

Applied to `<main>` or `.nb-article-body` on page load. The Binding Thread's vertical line + nodes become the visual axis that the clip-path reveals outward from.

### 4c-1. 2D Canvas Progress Charts

A lightweight `<canvas>` element with 2D rendering context for COI cost projection charts, IRR comparison graphs, and premium breakdown visualizations. Same technique Thorgal uses for its loader ring — pure Canvas 2D API, no WebGL context, no shader compilation overhead.

```ts
interface ChartConfig {

}
function drawChart(canvas: HTMLCanvasElement, config: ChartConfig): void;
```

The `'sketch'` style adds slight jitter to line points for a hand-drawn feel.

## What We Explicitly Chose NOT To Build

| Feature | Reason |
|---|---|
| Dark mode toggle | Parchment IS the brand. Toggle undermines identity. |
| Dust Jacket overlay | Hero already communicates the brand thesis. Click-through overlay on top of a working Hero is friction without gain. |
| Search modal / Cmd+K | Requires JS on every page. `/search/` page with Pagefind preserves zero-JS default. |
| Tags as public routes | Tags are frontmatter metadata for build-time filtering, not public `/tags/<tag>/` pages. Revisit at 100+ articles. |
| Comments / reactions | Noise and incentive misalignment. |
| Newsletter signup | Anti-sales doctrine. RSS is the subscription mechanism. |
| Author pages | Single voice. The site IS the author. |
| Social share buttons | No social presence to link. Readers copy URLs. |
| Reading Pulse | Chose Binding Thread instead — navigational utility > atmospheric decoration. The Phase 2d clip-path polygon reveal serves as the visual companion to the Binding Thread's navigational role. |
| "The Unfinished" (`/open`) | Contradicts "เน้นเนื้อๆ ไม่เอาน้ำ" — publishing answers without conclusions is explicitly water. |
| Contact form | Static site. Email is intentional act that filters. |
| Animated hero banners | The creed is the hero. The Phase 1b session-aware hero skip is about *removing* animation on return visits, not adding more. |
| "Related articles" algorithm | Editorial linkage or nothing. |
| Manual tombstone gaps | Manual `removed_slugs.ts` maintenance is น้ำ at this scale. |
| Format in URLs | Format is visual metadata — keeps URLs clean and pillar-signalling. |
| Thai URLs | English kebab-case for copy-paste reliability in LINE, email, mobile. |
| WebGL / Three.js / React Three Fiber | Adds hundreds of KB to bundle. CSS paper grain overlay achieves same tactile result at zero bytes. Canvas 2D (Phase 4c) handles chart rendering without WebGL overhead. |
| GSAP / ScrollTrigger / Lenis | All proposed effects are achievable CSS-only or with micro vanilla JS. No animation library dependency. Both thorgal.com and pixila.net ship zero GSAP — we follow their lead. |
| Character-level text reveal | Splitting headings into per-character `<span>` elements adds significant DOM weight. The existing `inkSettle` blur-reveal keyframe achieves the same pacing control with one class on one element. |
| Taxi.js / Barba.js / Swup page transitions | Heavy library dependency. CSS `clip-path` reveals (Phase 2d) and Astro's native View Transitions API provide transition mechanics with zero added libraries. |
