## 🎯 SECTION 1: BRAND IDENTITY & PHILOSOPHY

### The Core Shift
**FROM:** "Camera" (Photorealistic 3D renders, neon, futuristic)
**TO:** "Canvas" (Flat vector illustration, storytelling, character-driven)

### The Philosophy
> **"We don't just show data. We show Nart fixing it."**

Every image tells a story. The mascot (Nart) is always **doing an active verb** — not standing idle. The style communicates: "We make complex things simple."

### Why This Works for Financial Content
| Old Style Problem | New Style Solution |
|-------------------|-------------------|
| 3D/Futuristic = Cold, intimidating | Flat Vector = Warm, approachable |
| Abstract shapes = Confusing | Character actions = Storytelling |
| Neon glow = Generic "AI art" | Matte finish = Intentional design |
| No personality = Forgettable | Mascot = Memorable brand anchor |

## 🧬 SECTION 3: THE "NART" MASCOT DNA

### 3.1 Core Character Specification

**PASTE THIS INTO EVERY PROMPT FEATURING A CHARACTER:**

> **"The Mascot:** A stylized, cute vector character named **'Nart'**. He has a round friendly face, short dark hair, and wears **distinctive large round glasses** (The 'Nerd' signal). He is wearing a **Teal Blue hoodie/shirt** (`#2bb1bb`). He has a cheerful, confident expression. The art style is flat, 2D vector, clean shapes, no outlines, similar to Namecheap or Headspace corporate art."

### 3.2 Character Rules (Non-Negotiable)

| Rule | Requirement | Why |
|------|-------------|-----|
| ✅ ALWAYS | Wears **Teal** hoodie/shirt | Brand uniform = instant recognition |
| ✅ ALWAYS | Wears **round glasses** | "Nerd" signal = brand name anchor |
| ✅ ALWAYS | **Doing something** (active verb) | Storytelling > static poses |
| ✅ ALWAYS | **Confident/helpful** expression | He solves problems, not creates them |
| ❌ NEVER | 3D rendered or photorealistic | Breaks the "Canvas" philosophy |
| ❌ NEVER | Standing idle with no context | Wastes storytelling opportunity |
| ❌ NEVER | Confused or distressed expression | He is the expert, not the victim |

## 🖼️ SECTION 6: THREE VISUAL MODES

### MODE 1: NARRATIVE VECTOR (Social Media) — 80% of content

**Best For:** Facebook, Instagram, Line Voom, Hooks, Carousels
**Vibe:** Namecheap/Headspace — Friendly, active, storytelling
**Key Feature:** Nart doing an active verb with branded props

**Technical Specs:**
- Style: Flat 2D vector, no outlines, clean shapes
- Finish: Matte, smooth, NO gloss/neon
- Composition: 40% Character + 60% Text/Negative space
- Background: Scene Template (not just solid color)

---

### MODE 2: MODERN MINIMALIST (Professional) — 15% of content

**Best For:** Website headers, PDF covers, Client proposals, Whitepapers
**Vibe:** Stripe/Apple — Clean, elegant, sophisticated
**Key Feature:** Abstract geometric shapes, NO mascot (or very subtle)

**Technical Specs:**
- Style: Flat vector with soft gradients
- Finish: Premium, generous negative space
- Composition: Centered text, balanced geometry
- Background: Navy with Teal/Amber abstract accents

---

### MODE 3: TECHNICAL DATA (Evidence) — 5% of content

**Best For:** Deep Dives, Reports, Charts, Comparisons, Legal documents
**Vibe:** Bloomberg/Economist — Precise, data-first, credible
**Key Feature:** Clear charts, distinct labels, NO mascot

**Technical Specs:**
- Style: Clean data visualization
- Finish: High contrast, readable
- Composition: 80% Chart + 20% Title
- Background: Light (White or `#ECEFF1`) for readability

| Platform | Format | Aspect Ratio | Dimensions | Notes |
|----------|--------|--------------|------------|-------|
| **Facebook Feed** | Landscape | 16:9 | 1200 x 675 px | Hero images, link previews |
| **Facebook Feed** | Square | 1:1 | 1080 x 1080 px | Maximum engagement |
| **Instagram Feed** | Square | 1:1 | 1080 x 1080 px | Standard posts |
| **Instagram Story** | Vertical | 9:16 | 1080 x 1920 px | Stories, Reels covers |
| **Carousel Slide** | Square | 1:1 | 1080 x 1080 px | Multi-slide posts |
| **Website Hero** | Wide | 21:9 | 2100 x 900 px | Homepage banners |
| **Blog Thumbnail** | Landscape | 16:9 | 1200 x 675 px | Article previews |
| **OG Image** | Landscape | 1.91:1 | 1200 x 630 px | Social sharing |

### Text Placement Rules

| Zone | Content | Size | Position |
|------|---------|------|----------|
| **Primary** | Thai Headline | Large, Bold | Top 30% or integrated with scene |
| **Secondary** | (English Subtitle) | Medium | Below Thai, lighter weight |
| **Labels** | Object labels | Small-Medium | On props, clear contrast |
| **Footer** | Brand mark (optional) | Small | Bottom corner, subtle |

### Safe Zones
- Keep critical text **15% away from edges**
- Mobile crop: Assume **center 80%** is safe
- Never place text over Nart's face

## 🔄 SECTION 10: MIGRATION PROTOCOL

### Existing Article Image Replacement Plan

**Phase 1: Audit (Day 1)**
1. List all published articles with images
2. Categorize each image by content type (Growth/Protection/Analysis/etc.)
3. Note aspect ratios currently in use
4. Prioritize: Flagship articles first, then by traffic

**Phase 2: Template Mapping (Day 2)**
| Article Type | Old Style | New Template | New Pose | New Prop |
|--------------|-----------|--------------|----------|----------|
| Unit-Linked | 3D Charts | Template A | The Pilot | Rocket |
| Medical Audit | Abstract shapes | Template C | The Detective | Magnifying Glass |

| Hidden Fees | Photorealistic | Template C | The Mechanic | Wrench |
| Tax Strategy | 3D Blueprint | Template E | N/A (Minimalist) | N/A |

**Phase 3: Generation (Days 3-5)**
1. Generate new images using templates
2. Quality check against checklist
3. Regenerate if rejection criteria met
4. Export in correct aspect ratios

**Phase 4: Replacement (Days 6-7)**
1. Update Hygraph CMS with new images
2. Clear CDN cache
3. Verify display on all platforms
4. Archive old images (don't delete yet)

**Phase 5: Verification (Day 8)**
1. Review all articles live
