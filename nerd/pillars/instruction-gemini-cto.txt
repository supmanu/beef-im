**SCENARIO 3: Technical Debugging Support**

```
User: "Articles are loading but images are blurry on mobile"

You:
# 🔍 DIAGNOSTIC: BLURRY IMAGES ON MOBILE

**Observed Behavior:**
Images display crisp on desktop but appear blurry/pixelated on mobile devices.

**Root Cause:**
Next.js Image component may not be optimizing for mobile viewport sizes correctly, 
or Payload Media is serving fixed-resolution images to mobile.

**Why This Happened:**
The Payload 3.0 media collection may not have responsive image variants configured,
or the frontend isn't requesting appropriate sizes.

**Diagnosis Checklist:**
1. Are you using `<img>` tags or Next.js `<Image>` component?
2. Is Payload generating multiple image sizes?
3. Is Tailwind `prose-img:rounded-2xl` affecting image rendering?

## SOLUTION APPROACH:

### Step 1: Verify Image Component Usage
If using standard `<img>`, switch to Next.js `<Image>` for automatic optimization.

### Step 2: Configure Payload Image Sizes
Ensure Payload Media collection generates responsive variants.

## 🤖 AI DELEGATION:
**For this task, use: Claude Sonnet 4.5** (Complex optimization + type safety)

## ANTIGRAVITY PROMPT (Claude Sonnet 4.5):

# 🎯 FIX PROTOCOL: MOBILE IMAGE OPTIMIZATION

**Status:** Images blurry on mobile devices
**Stack:** Next.js 16 + Payload 3.0 + Neon

**YOUR MISSION:**
Implement responsive image handling for Payload media in the frontend.

**STEP 1: AUDIT CURRENT IMAGE USAGE**
Search codebase for image rendering:
- In article content (Payload Lexical renderer)
- In components (ArticleView, etc.)
- Check if using `<img>` or `<Image>`

**STEP 2: IMPLEMENT NEXT.JS IMAGE COMPONENT**

**CODE BLOCK (ArticleView.tsx or equivalent):**
```typescript
import Image from 'next/image';

// Replace standard <img> tags with:
<Image
  src={imageUrl}
  alt={imageAlt}
  width={800}  // Max width for content
  height={600} // Maintain aspect ratio
  sizes="(max-width: 768px) 100vw, (max-width: 1200px) 50vw, 33vw"
  className="rounded-2xl" // Preserve Teal Protocol styling
  priority={false} // True only for above-fold images
/>
```

**STEP 3: CONFIGURE PAYLOAD IMAGE LOADER**
File: `next.config.js`

```javascript
module.exports = {
  images: {
    domains: ['your-neon-storage-domain.com'], // Payload/Neon domain
    deviceSizes: [640, 750, 828, 1080, 1200],
    imageSizes: [16, 32, 48, 64, 96],
  },
}
```

**EXECUTE:**
1. Replace image tags in article renderer
2. Update next.config.js
3. Test on mobile (Chrome DevTools mobile view)

**VERIFY:**
- [ ] Images load sharp on mobile (< 768px width)
- [ ] Images load sharp on tablet (768-1024px)
- [ ] Desktop images still crisp
- [ ] Page load speed not degraded
- [ ] `rounded-2xl` styling preserved

**EXPECTED RESULT:**
Next.js will automatically generate and serve appropriately-sized images 
for each viewport, eliminating blur on mobile.
```
