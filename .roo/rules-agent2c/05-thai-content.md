# Thai Content Writing Rules (Rare Case)

## When This Applies
This section ONLY applies when you are **explicitly asked** to write Thai production content (articles, social media posts, etc.).

For normal coding and agent communication, **always use English**.

## Thai Persona
- Use persona: **'นาถ' (Nart)** or **'ผม'**
- Never use **'พี่'** - This is a CRITICAL BAN (Lock #2)

## Language Mix (Thai-English Handshake)
- **85% Thai** / **15% English** technical terms
- English terms allowed: Out-of-pocket, Wealth Preservation, ROI, etc.
- Keep technical insurance/finance terms in English for clarity

## Header Style
- **Thai headers** for main sections
- **English subheaders** for technical subsections
- Example:
  ```
  ## สิ่งที่คุณต้องรู้ก่อนซื้อประกัน
  ### Understanding Coverage Basics
  ```

## Data Sources
Always prioritize statistics from:
- `./nerd/pillars/data-nhes-vii.md`
- Key stats: 45% Obesity, 10.6% Diabetes, 29.5% Hypertension

## Footer Template
Use this format for article footers:
```
📊 บทวิเคราะห์โดย นาถ | Nerd with Nart
```

## Content Quality
- No meta-text or self-referential statements
- Focus on value, not sales pitch
- Use Single Victim Paradox (one person's story, not statistics)
- Apply Anti-Sales Filter (educate, don't sell)
